# AIDC技术选型核心逻辑 - 行业关键洞察

## 核心观点（2026-05-13 13:36）

**AIDC的技术选型核心与甲方的算力资源和算力卡厂商有很大关系，这是一个行业核心问题。**

---

## 深度解析

### 一、技术选型的三个关键变量

#### 1. 甲方现有的算力资源
- **历史投资**：已有多少GPU/CPU？
- **平台锁定**：英伟达CUDA生态？华为CANN生态？
- **迁移成本**：切换平台的成本有多高？

#### 2. 甲方选择的算力卡厂商
- **英伟达**：性能领先，价格高，800V HVDC刚需
- **华为昇腾**：性价比高，国产化，传统供配电
- **混合方案**：训练用英伟达，推理用昇腾

#### 3. 供配电系统的适配性
- **英伟达路线**：需要800V HVDC + 液冷
- **华为路线**：传统供配电 + 液冷
- **混合路线**：需要同时支持两种架构

---

## 二、技术选型的决策树

### 场景1：甲方从零开始新建AIDC

**关键问题：**
1. **预算充足？**
   - 是 → 英伟达路线（800V HVDC + 液冷）
   - 否 → 华为昇腾（传统供配电 + 液冷）

2. **国产化要求高？**
   - 是 → 华为昇腾优先
   - 否 → 英伟达性能优先

3. **功率密度要求？**
   - 单机柜100kW+ → 英伟达 + 800V HVDC
   - 单机柜50kW以下 → 华为昇腾 + 传统供配电

### 场景2：甲方已有算力资源，扩容/升级

**关键问题：**
1. **现有平台是什么？**
   - 英伟达CUDA生态 → 继续英伟达（迁移成本高）
   - 华为CANN生态 → 继续昇腾

2. **扩容功率密度？**
   - 维持现有密度 → 继续现有方案
   - 提升密度 → 考虑800V HVDC

3. **混用场景？**
   - 训练用英伟达（高功率）
   - 推理用昇腾（性价比）

---

## 三、行业核心问题的本质

### 为什么是"行业核心问题"？

**1. 决定了整个供配电市场格局**
- 英伟达路线 → 维谛、伊顿等800V HVDC厂商机会大
- 华为路线 → 华为全栈（FusionPower6000 + 液冷）
- 混合路线 → 需要兼容性方案

**2. 决定了技术演进方向**
- 英伟达定义800V HVDC → 全行业跟进
- 华为昇腾不需要800V → 传统供配电仍有市场
- SST/巴拿马电源等新技术路线分化

**3. 决定了厂家的竞争策略**
- 维谛：与英伟达深度合作，800V HVDC先发
- 华为：垂直整合，算力+供配电+液冷全栈
- 伊顿：跟进800V HVDC，但相对慢

## 四、对东方物探的启示

### 东方物探的算力资源现状
- **已与华为深度合作**（2023年10月启动）
- **鲲鹏平台适配**（2000万行代码已移植）
- **昇腾AI加速**（全波形反演效率提升数倍）
- **GeoEast系统**（3000万行代码，自主知识产权）

### 技术选型的可能路径

**路径1：继续华为路线（概率高）**
- **理由**：
  - 已有大量投资（2000万行代码迁移）
  - 信创压力（央企国产化要求）
  - 昇腾效率提升（2-3倍）
- **供配电**：华为FusionPower6000 + 液冷
- **维谛机会**：⭐⭐（华为自己做全栈）

**路径2：英伟达路线补充（概率中）**
- **理由**：
  - 部分AI训练需要英伟达GPU
  - 超高功率密度场景
- **供配电**：800V HVDC + 液冷
- **维谛机会**：⭐⭐⭐⭐（英伟达部分）

**路径3：混合路线（概率中高）**
- **理由**：
  - 训练用英伟达（高功率）
  - 推理用昇腾（性价比）
- **供配电**：需要同时支持两种架构
- **维谛机会**：⭐⭐⭐（兼容性方案）

## 五、对OPC产品的价值

### 核心预测维度

**1. 算力平台选择**
- **数据源**：
  - GPU/TPU采购公示
  - 与华为/英伟达的合作新闻
  - 招标文件的技术规格要求
  - 算力中心投资计划

- **预测逻辑**：
  - 甲方现有算力资源是什么？
  - 信创压力大不大？
  - 功率密度要求多高？
  - → 判断选英伟达还是昇腾

**2. 功率密度预测**
- **数据源**：
  - 机房改造/新建计划
  - 算力需求增长趋势
  - 历史采购记录（功率密度变化）

- **预测逻辑**：
  - 当前机柜功率密度是多少？
  - 增长趋势如何？
  - → 判断什么时候需要液冷
  - → 判断什么时候需要800V HVDC

**3. 供配电需求匹配**
- **数据源**：
  - 算力平台选择 + 功率密度
  - 现有供配电系统状况

- **预测逻辑**：
  - 选英伟达 + 高功率密度 → 推荐800V HVDC厂商（维谛、伊顿）
  - 选华为昇腾 → 推荐华为全栈或兼容方案
  - 混合路线 → 推荐兼容性方案

## 六、行业洞察

### 技术路线的分化趋势

**2024-2026年关键时间窗口：**
- **2025年**：英伟达发布800V HVDC白皮书
- **2025年10月**：维谛宣布与英伟达深度合作，2026H2量产800V HVDC
- **2025年**：伊顿推出800V架构
- **2026年**：华为昇腾持续迭代，但不需要800V HVDC

**行业分化：**
- **英伟达阵营**：维谛、伊顿等供配电厂商，围绕800V HVDC
- **华为阵营**：华为全栈（算力+供配电+液冷）
- **兼容阵营**：同时支持两种架构的厂商

### 厂家竞争格局

**维谛：**
- 优势：与英伟达深度合作，800V HVDC先发
- 劣势：华为全栈在昇腾场景下优势明显
- 策略：主推800V HVDC + 液冷一体化

**华为：**
- 优势：昇腾不需要800V HVDC，垂直整合能力强
- 劣势：在英伟达路线场景下难以切入
- 策略：全栈方案（算力+供配电+液冷+网络）

**伊顿：**
- 优势：2025年跟进800V HVDC
- 劣势：液冷/模块化相对保守
- 策略：跟随维谛，强调可靠性

## 七、关键结论

### 对桥哥的核心洞察

**1. AIDC技术选型的核心是算力平台选择**
- 不是选液冷还是风冷
- 不是选UPS还是HVDC
- 而是**先选算力卡厂商，再选供配电方案**

**2. 这验证了OPC产品的核心假设**
- 需要预测甲方的算力平台选择
- 算力平台选择 → 决定供配电供应商
- 这是商机预测的关键维度

**3. 数据挖掘的重点方向**
- 算力平台选择倾向（英伟达 vs 华为）
- 功率密度预测（什么时候需要液冷/800V）
- 供配电需求匹配（推荐对应的供应商）
