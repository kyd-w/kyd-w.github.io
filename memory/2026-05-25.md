# 混合数据源方案设计

时间: 2026-05-25 11:35
目标: 结合 RSS 和新闻聚合 API，优化 IT 和 AI 行业章节

## 数据源分层策略

### 第一层：RSS 源（70-80%）
**IT 行业**：
- IT之家: https://www.ithome.com/rss/
- 36氪: https://36kr.com/feed
- InfoQ: https://www.infoq.cn/feed
- 虎嗅: https://www.huxiu.com/rss/0.xml

**AI 行业**：
- 机器之心: https://www.jiqizhixin.com/rss
- IT之家 (AI 标签)
- 36氪 (AI 标签)
- Quantum Intelligence: https://www.quantamagazine.org/feed/

**优势**：
- ✅ 权威媒体，质量高
- ✅ 天然时效性（按时间排序）
- ✅ 内容完整（标题、摘要、链接）

### 第二层：新闻聚合 API（10-20%）
**可选服务**：
- NewsAPI: https://newsapi.org/
  - 免费版：100次/天
  - 支持中文（有限）
  - 可按类别、语言、时间过滤

- GNews: https://gnews.io/
  - 免费版：100次/天
  - 支持中文
  - 搜索特定主题

- Bing News API
  - 需要 Azure Key
  - 覆盖面广

**用途**：
- 补充国际新闻（TechCrunch、VentureBeat）
- 获取特定技术话题的深度报道
- RSS 源未覆盖的领域

### 第三层：搜索引擎（0-10%，备选）
- Tavily 搜索
- 仅在 RSS 和 API 都不足时使用
- 搜索具体技术话题（如 "Rust 编程"、"Web3"）

## 智能去重策略

### 三级去重机制

**1. 标题去重**
```python
# 标题相似度 > 85% 视为重复
normalized_title = remove_punctuation(title.lower())
if normalized_title in seen_titles:
    skip
```

**2. URL 域名去重**
```python
# 同一域名的新闻只保留最新的
domain = urlparse(url).netloc
if domain in latest_domains and date <= latest_domains[domain]:
    skip
```

**3. 内容指纹去重**
```python
# 前 100 字的相似度
fingerprint = summary[:100].lower()
if similarity(fingerprint, existing) > 0.8:
    skip
```

## 内容质量控制

### 过滤规则

**时间过滤**：
- IT 行业：≤ 2 天
- AI 行业：≤ 3 天（AI 发展快，新闻更持久）

**质量过滤**：
- 最小长度：标题 ≥ 10 字，摘要 ≥ 30 字
- 关键词匹配：必须包含行业相关词
- 来源白名单：只接受权威媒体

**排序规则**：
1. 按时间倒序
2. 同一天内按来源优先级排序
3. 优先选择有具体数据和事件的文章
