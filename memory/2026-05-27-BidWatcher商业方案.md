# BidWatcher 数据中心/AIDC 垂直行业情报平台 — 商业方案

> 版本：v3.0（产品方向调整：ToB市场情报平台 — 目标用户→设备供应商市场/BD部门）
> 日期：2026-05-26
> 状态：Phase C 已完成，Phase D 验证期启动
> 收录日期：2026-05-27

## 核心要点记录

### 产品定位
- **一句话**：数据中心/AIDC基础设施行业的垂直情报平台
- **目标用户**：设备供应商（维谛、华为、施耐德等）的市场/BD部门（非一线销售）
- **核心价值**：L0~L8九级生命周期追踪，提前4-14月预测设备采购窗口
- **关键转变**：从"帮销售找项目"→"帮市场团队理解行业运行规律"

### 生命周期模型（L0~L8）
- L0：项目审批（最早信号）
- L1：环评审批
- L2：用地+规划
- L3：设计招标
- L4：施工总包招标（总包公司核心节点）
- L4.5：施工许可
- L5：基础设施设备招标（UPS/精密空调/配电等 - 维谛核心节点）
- L6：IT设备招标（GPU服务器/交换机/存储）
- L7：竣工验收
- L8：扩容/运维

### 四类建设场景
- A：普通建筑→AIDC改造（14-16月）
- B：空地→AIDC新建（24-26月，信息最丰富）
- C：传统IDC→AIDC升级（8-10月）
- D：现有AIDC扩容（4-6月）

### 竞品差异化
- vs 知了标讯（10-100万/年）：拼垂直深度，不拼覆盖面
- vs 企查查/乙方宝：拼分析洞察，不拼数据量
- vs IDC/艾瑞：拼实时性和颗粒度（月度vs年度，项目级vs宏观）

### 核心能力
1. **项目链路预测**（第一层）：L0→L8节点推进规律
2. **设备周期预测**（第二层，待建）：基于设备物理寿命（UPS电池3-5年等）
3. **AI结构化提取**：M0-M9模块化提取
4. **总包→分包组织架构分析**
5. **算力平台选择预测**：英伟达vs华为路线决定供配电方案

### 数据来源
- 当前：政府采购网（ccgp.gov.cn）+ 中国通用招标网 + 招标投标公共服务平台
- 覆盖率：约30-40%（政府采购类）
- 公告层信息足以支撑"市场情报"定位
- 盲区：央企自有采购平台（70+家，35%最大盲区）

### 采集范围（基础设施全栈）
1. 供配电系统（UPS/配电柜/发电机/蓄电池等）
2. 制冷系统（精密空调/冷水机组/液冷）
3. 机柜与物理设施
4. 消防与安全
5. 管理与监控系统（DCIM/动环监控）
6. 工程服务

### 商业模式
- 阶段1（当前）：人工验证，拿报告找3-5个目标用户聊
- 定价：月报¥2,000-5,000/月，定制情报¥10,000-30,000/次，年度订阅¥50,000-100,000/年
- 远低于知了标讯（10-100万/年）

### 成功标准（Phase D验证期）
- L0~L8分类覆盖率 ≥ 80%
- 跨节点项目关联：180天窗口内 ≥ 10个多阶段项目
- AI提取准确率 ≥ 85%
- 市场团队认可：≥2人认可"有用"
- 付费意愿：≥1人愿意付费

### 市场验证记录
- 2026-05-13：东方物探拜访（维谛销售孟诚）
- 验证结果：认可"前置信息"和"关键人识别"价值
- 方向调整：从销售需求转向市场团队需求

### 关键洞察
- **招标节点是价值体现节点，不是参照节点**：同一项目的情报对不同受众价值完全不同
- **公告层信息支撑市场情报，不支撑投标辅助**（参数/品牌/评分在标书层）
- **设备周期预测 vs 统计预测**：物理规律>统计规律（"设备该换了" > "好像每年都买"）
